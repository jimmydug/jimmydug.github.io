上例中i += x和i = i+x出现了不一致的结果。原因下面来讨论：
Python中的对象大体分为俩类：

不可变对象

不可变对象(immutable)有int，str，tuple，set，特点是对象在内存中的值永远是刚创建时的值，任何操作都会创建一个新的对象而不是改变1原对象。

对int型不可变对象1做的操作都创建了一个新的对象，变量a也指向了新对象。

可变对象

可变对象(mutable)有dict，list，特点是对象值可以动态的改变，当赋值给俩个变量时，俩个变量指向同一个对象，对一个变量操作时会影响另一个变量。

进行append操作时，对象的内存地址不改变，俩个变量仍指向这个对象，只是对象的值发生了变化。

回到话题，+=和+的区别是什么呢

+=操作首先会调用对象的__iadd__()方法，如果没有该方法那就会调用__add__()方法，这俩个方法是有区别的，__iadd__()接受俩个参数，且会改变第一个参数的值，这需要第一个参数是可变对象，故不可变对象没有__iadd__()方法，__add__()方法也接受俩个参数返回一个新的对象，而不改变原俩个对象的值。

现在可以解释刚开始那个现象了，list是可变对象，有__iadd__()方法，a += [9]操作中调用了__iadd__()改变了原对象的值，变量a指向不变，故b也变化，a=a+[9]中调用了__add__()方法，返回了一个新对象，变量a指向新对象，而b依旧指向老对象，故b值不变。
